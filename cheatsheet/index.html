Build WordPress sites with Claude — section by section.

Get a Figma design and a task → paste the URL into Claude → pixel-perfect WP section with ACF, ready to seed. Loop continuously, retro at the end, push improvements to GitHub for the next project.

First-time setup on a new project

One time per project. ~2 minutes total. Then you're ready to build.

1

Generate underscoretw theme

Go to underscoretw.com → fill in name + slug + function prefix → download zip → extract into wp-content/themes/<theme>/ → activate in WP admin.

2

Install npm + run Tailwind watcher

npm install
npm run watch

Leave it running. If PC slows, watcher pauses silently → restart.

3

Drop the template into your theme

From GitHub (recommended)

cd <your-project>/wp-content/themes/<theme>
git clone --depth 1 https://github.com/KuyaLoy/wp-claude-template.git .claude-tmp
cp -r .claude-tmp/claude-setup ./.claude
rm -rf .claude-tmp

Or local copy (Windows / Mac)

# Windows
xcopy /E /I "<path-to-master>\claude-setup" "<your-project>\wp-content\themes\<theme>\.claude"

# Mac / Linux
cp -r <path-to-master>/claude-setup <your-project>/wp-content/themes/<theme>/.claude

⚠️ Folder name should be .claude (dot prefix). Hidden by default — Windows: enable "Hidden items". Mac: ⌘+Shift+.

4

Run /setup-claude

/setup-claude

Claude auto-detects project name, theme slug, local URL. Asks 2-3 questions:

  • Project name correct?
  • Figma URL? (for auto-detecting colors/font)
  • Anything special? (phone, container, breakpoints)
5

Configure WP admin (~1 min)

  1. Settings → Permalinks → "Post name" → Save
  2. Pages → Add New → "Home" → Template = Homepage → Publish
  3. Settings → Reading → Static page → Homepage = Home
  4. Custom Fields → Field Groups → Sync changes

🔄 Reset and start over?

Say in Claude: "reset my project so I can re-run /setup-claude". Claude wipes .claude/, briefs/, the personalized README, and the setup-generated files. Then re-do steps 3-5.

How you actually prompt Claude

Day-to-day, you receive a task + a Figma link. Paste them into Claude. That's it.

📋 The realistic prompt

Build the services-grid section from this Figma:
https://figma.com/design/abc?node-id=5404-718

Tweaks (deviations from Figma):
- Use one whole image as the bg instead of per-card images
- Cards stack vertically on mobile (Figma only shows desktop)
- h3 for card titles, not h2
- Add "Learn more" link below each card linking to /service/

No file to create, no special syntax. Free-form description + Figma URL + bullet list of tweaks. Works the same in Claude Cowork (chat) and Claude Code (VSCode).

What to include in your prompt

  • What: section name + Figma URL with node-id for the specific frame
  • Mobile: if Figma has a mobile frame, paste BOTH URLs. If not, Claude makes sensible defaults and tells you
  • Tweaks: deviations from Figma — bg images, heading levels, animations, links, etc.
  • Context: "match the services-grid I built last week", "use the icon style from rope-access", etc.
⚙️ Optional: structured brief file (handoff / complex sections only)

For team handoffs or sections with 10+ ACF fields, write a brief at briefs/<name>.md (template at briefs/_template.md). Then:

# Claude Cowork
@briefs/services-grid.md https://figma.com/design/abc?node-id=5404-718

# Claude Code
/implement briefs/services-grid.md https://figma.com/design/abc?node-id=5404-718

For solo dev work, skip this — the inline prompt above is faster.

The 5-step workflow (the bible)

Every section in the codebase was built via these 5 steps in this order. Don't skip steps; don't merge them.

1

Static

Hard-code from Figma. Pixel-perfect. No ACF.

2

Cross-check

Live render vs Figma. Flag & fix deviations.

3

Dynamic

Strings → get_sub_field(). Write ACF JSON.

4

Sync

WP admin → ACF → click Sync.

5

Seed

One-shot self-deleting PHP seeder.

Why this order: static-first catches design issues before ACF complexity makes them harder to fix. Self-deleting seeders are versioned, reproducible migrations.

Sample prompts — cookbook

Copy-paste these. Edit the specifics. Works in both ⚡ CW Cowork and 💻 CC Claude Code.

📐 Build a section

Build the hero section from this Figma:
https://figma.com/design/.../?node-id=5404-718

Mobile: https://figma.com/design/.../?node-id=5404-720

Tweaks:
- Full-bleed bg image
- h1 not h2
- Primary CTA = "Get a Quote" (#quote = opens modal)

🔄 Make section dynamic

Make services-grid dynamic. Add ACF fields:
- heading (text)
- tagline (text)
- items (repeater): title, body, icon (textarea-svg), link
- bg_variant (true/false)

After, write a seeder to populate 6 items on the Service page.

🔍 Audit all CTAs

Check every CTA on every page. List page, label, target URL. Flag any that point to "#" or are empty. Suggest #quote for quote-flavoured buttons.

🧭 Rebuild navigation menu

Write a one-shot seeder at inc/seed-menu-rebuild.php that wipes the Main Menu and re-adds these in order: Painting, Rope Access, Working at Heights, Projects, About Us, Careers. Assign to Primary location.

📦 Add a Custom Post Type

Create a "Project" CPT. Archive slug: /projects/. Taxonomy: project_category. Supports: title, editor, thumbnail, excerpt, page-attributes. Menu icon: dashicons-portfolio. Use ACF JSON sync.

📨 CF7 form + branded email + redirect

Create CF7 form "Request a Free Quote" with fields: first-name, last-name, email, phone, postcode, service (dropdown), message. Configure Mail to admin@example.com with BCC, branded HTML email. Per-form redirect to /thank-you/ on success.

✏️ Rename page (keep URL)

Rename "Service" page to "Painting". Change Page Title + menu label only. Keep slug /service/ to preserve SEO. Use a one-shot seeder.

🔀 Rename page (change URL with 301)

Rename "Service" → "Painting". Change slug /service/ → /painting/. Add 301 redirect /service/ → /painting/ for SEO.

🏁 Pre-staging check

Run a final pre-staging audit. Check: every page returns 200, no PHP errors, no broken # CTAs, all phones consistent, CF7 has HTML emails, modal opens on every page. Report in a table.

↩️ Revert a mistake

Show me git status. Revert the last change to file X. If it's ACF JSON, I'll re-sync in admin.

🛠 Tailwind class not showing

The class `lg:max-w-[680px]` isn't applying on the modal. DON'T inline CSS yet — first check: is my `npm run watch` running? If yes, what's the computed style?

⚠️ v3 rule: Claude asks about the watcher BEFORE inlining CSS.

ACF patterns — copy/paste snippets

Every dynamic section uses these. Always escape output. Always guard with if.

Text field

$heading = (string) ( get_sub_field( 'heading' ) ?? '' );
if ( trim( $heading ) !== '' ) : ?>
    <h2><?= esc_html( $heading ) ?></h2>
<?php endif;

Image field (ACF array)

$img = get_sub_field( 'image' );
if ( $img ) : ?>
    <img src="<?= esc_url( $img['url'] ) ?>"
         alt="<?= esc_attr( $img['alt'] ) ?>"
         width="<?= (int) $img['width']  ?>"
         height="<?= (int) $img['height'] ?>"
         loading="lazy" decoding="async">
<?php endif;

Repeater loop

if ( have_rows( 'items' ) ) : while ( have_rows( 'items' ) ) : the_row();
    $title = get_sub_field( 'title' );
    if ( ! $title ) continue;
    ?>
    <article><h3><?= esc_html( $title ) ?></h3></article>
<?php endwhile; endif;

Options page field

$phone = (string) get_field( 'phone', 'option' );

SVG textarea (sanitized)

$svg = get_sub_field( 'icon_svg' );
if ( $svg ) : ?>
    <div class="w-12 h-12 text-primary">
        <?= wp_kses( $svg, aiims_svg_kses() ) ?>
    </div>
<?php endif;

Link field

$link = get_sub_field( 'cta' );
if ( $link && ! empty( $link['url'] ) ) : ?>
    <a href="<?= esc_url( $link['url'] ) ?>"
       target="<?= esc_attr( $link['target'] ?: '_self' ) ?>">
        <?= esc_html( $link['title'] ) ?>
    </a>
<?php endif;

Seeder template — one-shot, self-deleting

Save to inc/seed-<feature>.php, register in functions.php seeders array, trigger via /?aiims_seed=<slug> as admin.

<?php
/**
 * One-shot seeder — what this does.
 * Trigger (admin only): /?aiims_seed=<slug>
 * Self-deletes on success.
 */
add_action( 'init', function () {
    if ( ! is_admin() && empty( $_GET['aiims_seed'] ) ) return;
    if ( ( $_GET['aiims_seed'] ?? '' ) !== '<slug>' ) return;
    if ( ! current_user_can( 'manage_options' ) ) return;
    if ( ! function_exists( 'update_field' ) ) wp_die( 'ACF not active.' );

    // === DO THE WORK HERE ===

    $self    = __FILE__;
    @chmod( $self, 0666 );
    $deleted = @unlink( $self );

    wp_die(
        '<h2>Seeded.</h2><p>Seeder ' . ( $deleted ? '<strong>deleted</strong>' : 'NOT deleted' ) . '</p>',
        'Done', [ 'response' => 200 ]
    );
} );

Why self-deleting: seeders are migrations. Run once, gone. Versioned in git via the PHP file. Reproducible by recreating.

Images & SVG — decision tree

Three patterns for SVG; one helper for static raster.

Pattern A · Textarea

Editor pastes raw SVG. Use fill="currentColor". Sanitize with wp_kses().

Use for: brand-coloured icons, checkmarks, accents.

Pattern B · Image field

Editor uploads SVG/PNG file. Render as <img> with width/height.

Use for: brand logos with fixed colors, partner badges.

Pattern C · Inline

Hardcoded in template. Site-wide icons (chevron, close X).

Use for: theme-wide icons that never change.

Static raster (logo, banner)

<?php aiims_img('hero/banner.jpg', 'Alt text', 'w-full h-auto'); ?>

Auto-reads width/height from disk. Use for static theme assets only.

⚠️ Every <img> MUST have width and height attributes — required for CLS (Cumulative Layout Shift).

⚖️

Figma exactness vs best practice — when to override

A Figma design is a guide, not gospel. Sometimes you follow it exactly; sometimes you override for technical reasons. Here's the call:

✅ Follow Figma exactly when

  • The choice is purely visual: colours, spacing, typography, layout
  • The brand has rules about specific things (logo position, palette)
  • It's the designer's clear intent and there's no technical reason against

⚠️ Override for best practice (and FLAG it to the designer)

  • Accessibility: contrast ratio below WCAG AA → bump it; touch targets < 44×44px → enlarge
  • Semantic HTML: Figma shows visual h2 but document outline needs h1 → use h1
  • Performance: 30 different fonts/weights → consolidate to 2-3 weights of brand font
  • Responsive: Figma only shows desktop → add sensible mobile defaults, flag for review
  • SEO: Figma uses image-as-heading → use real h2 + image as decoration
  • Maintainability: Figma uses 8 different button styles → propose 3 + variants

❌ Don't override on whim

  • "I think this looks better" without a technical reason
  • "I'd do it differently on a personal project" — not your call
  • Container widths (1320px vs 1280px — silently rounding)
  • Brand colors (hex variations the eye can spot)

The rule: if you override, surface it in your reply ("Deviations from Figma: …"). The designer decides whether to accept the override or change Figma to match. Never silently deviate.

🚫

Things NOT to do

  • ✗Don't build full pages in one shot. Section-by-section.
  • ✗Don't skip the static phase. Always pixel-match Figma before ACF.
  • ✗Don't inline CSS as a Tailwind workaround without first confirming the watcher is actually running.
  • ✗Don't use display: none on mobile to hide design pieces — restructure.
  • ✗Don't silently round container widths. Surface deviations.
  • ✗Don't generate a new ACF key when editing an existing field. Preserve the original.
  • ✗Don't skip escaping. Every dynamic output: esc_html() / esc_attr() / esc_url() / wp_kses_post().
  • ✗Don't skip width and height on <img> tags — required for CLS.
  • ✗Don't add features the client didn't ask for (CPTs, search, etc. — only on request).
  • ✗Don't edit a completed project's .claude/. Move improvements upstream to wp-claude-template.
  • ✗Don't forget to push the template to GitHub after a retro.

After a project ships — run the retro

The whole point of this template: every project teaches you something. After ship → retro → improve template → push to GitHub → next project is better.

The loop

SHIP → snapshot to completed-projects/<name>/  →  open Claude on the template
         →  diff snapshot vs template  →  propose v(next) updates
            →  approve each one  →  bump CHANGELOG  →  commit  →  PUSH TO GITHUB
               →  next project clones the improved template

Detailed steps in RETRO-WORKFLOW.md.

Sample retro prompt:

I just finished a project at completed-projects/<name>. Diff that snapshot against the current claude-setup/. Identify what worked, what was painful, propose v<next> updates. Add a new section to RETRO.md. Wait for me to approve each change.

Troubleshooting

Tailwind class isn't applying

First — is npm run watch still running? Watcher silently pauses if PC stalls. Close terminal and restart:

npm run watch

Then hard-refresh: Ctrl+Shift+R / Cmd+Shift+R.

ACF "Sync available" won't go away

JSON modified timestamp newer than DB. Click Sync in WP admin. If it persists, bump the timestamp in the JSON file (touch will do).

Permalinks broken / inner pages 404

Rewrite rules not flushed. WP admin → Settings → Permalinks → Save Changes (don't change anything; saving flushes).

CF7 not sending mail / mail in spam

  1. Install SMTP plugin (WP Mail SMTP / FluentSMTP)
  2. SPF + DKIM DNS records for the domain
  3. Real SMTP provider (Mailgun, Postmark, SendGrid, Google Workspace)
  4. "From" header must be on same domain — never visitor's email

Quote modal not opening

CTA URL must be exactly #quote in ACF. Check dev tools console for JS errors. View source for id="quote-modal" in the footer.

Seeder says "not found" or skips silently

  • Target post slug changed / deleted
  • Not logged in as admin (requires manage_options)
  • Seeder PHP file already deleted (self-cleanup) — recreate from git
  • Not registered in functions.php seeders array

PHP fatal: memory exhausted

// wp-config.php
define( 'WP_MEMORY_LIMIT', '512M' );

Or update php.ini: memory_limit = 512M.

Project slug doesn't match title (URLs weird)

Happens after duplicate + rename. Edit post → click Permalink under title → change slug → Update. Or programmatically:

Fix slug for project ID X to "my-new-slug". One-shot seeder using wp_update_post + flush_rewrite_rules.

Git push: Permission denied / 403

Authenticated as wrong account. Windows Credential Manager → remove all git:https://github.com + gh:github.com:* entries → push again → enter correct username + PAT.

🆘 When in doubt — how to ask Claude

Give Claude all four:

  1. What you expected
  2. What actually happened (paste error message verbatim)
  3. What you already tried
  4. The URL or file involved

A complete bug report saves a 5-message back-and-forth.
