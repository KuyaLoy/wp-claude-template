Setting up Claude on a new project

One-time per project. ~2 minutes total.

📦 Where the template lives

<path-to-master> = wherever you have claude-setup/ (local drive, shared drive, Git repo, or a zip you got from a teammate). <your-project> = your WordPress project root. <theme> = the theme folder slug.

  1. 1

    Generate the underscoretw theme

    Go to underscoretw.com, fill in name + slug + function prefix. Download the zip, extract into wp-content/themes/<theme>/. Activate in WP Admin.

  2. 2

    Install npm + start Tailwind watcher

    npm install npm run watch

    Leave running.

  3. 3

    Copy claude-setup/ into your theme

    Windows / PowerShell

    xcopy /E /I "<path-to-master>\claude-setup" "<your-project>\wp-content\themes\<theme>\claude-setup"

    Mac / Linux

    cp -r <path-to-master>/claude-setup <your-project>/wp-content/themes/<theme>/claude-setup

    From a zip

    Unzip → drag the inner claude-setup/ folder into your theme.

    From Git

    cd <your-project>/wp-content/themes/<theme> git clone --depth 1 <repo-url> .claude-tmp cp -r .claude-tmp/claude-setup ./claude-setup rm -rf .claude-tmp
  4. 4

    Rename to .claude

    # Windows ren claude-setup .claude # Mac / Linux mv claude-setup .claude

    Dot-folders are hidden in some File Explorer / Finder views. Windows: turn on "Hidden items". Mac: ⌘+Shift+.

  5. 5

    Run /setup-claude

    /setup-claude

    Claude auto-detects project name, theme slug, local URL, author. Asks 2-3 short questions:

    1. Project name correct? (yes / type the right name) 2. Figma file URL? (auto-detect colors/font/container — or skip) 3. Anything special? (phone is US? container 1920? — or "default") Defaults: lg: breakpoint, AU phone, no Swiper, container 1320px
  6. 6

    Claude bootstraps everything (~30 sec)

    Files created:

    theme/inc/helpers.php
    theme/inc/acf-setup.php
    theme/inc/custom-functions.php
    theme/templates/template-homepage.php
    theme/templates/template-default.php
    theme/templates/parts/section-_example.php
    theme/acf-json/group_default_template_sections.json
    theme/assets/images/, theme/assets/icons/
    briefs/_template.md
    README.md (workspace root — your daily reference)

    Updated:

    theme/functions.php (one require added)
    tailwind/tailwind-theme.css (brand tokens)

    Self-deleted (cleanup):

    .claude/snippets/
    .claude/skills/setup-claude/
    .claude/commands/setup-claude.md
    .claude/README.md (internal)

    Stays: .claude/CLAUDE.md, .claude/skills/ (10 daily skills), .claude/agents/, .claude/commands/, .claude/cheatsheet/index.html (this page).

  7. 7

    Configure WP Admin (~1 min)

    1. Settings → Permalinks → Post name → Save
    2. Pages → Add New → "Home" → Page Attributes → Template = Homepage → Publish
    3. Settings → Reading → Static page → Homepage = Home
    4. Custom Fields → Field Groups → click Sync changes
  8. 8

    Build your first section

    Build the home hero from this Figma: <url>

    Claude asks 2-3 questions, scaffolds briefs/home-hero.md, builds the static section.

Need to reset and start over?

Just say: "reset my project so I can re-run /setup-claude". Claude wipes .claude/, briefs/, the personalized README, and everything in theme/inc/, theme/templates/, theme/acf-json/ that setup created. Then re-do steps 3-8.

The workflow

One section at a time. Two phases per section, no shortcuts.

Phase 1

Static

Hard-coded markup matching Figma. No ACF. Easy to verify pixel-perfect.

  1. (Optional) write briefs/foo.md
  2. Type: Build foo from <figma>
  3. Hard-refresh, eyeball
  4. Approve when pixel-perfect

Phase 2

Dynamic (ACF)

Same markup, content swapped for ACF. Schema in acf-json/.

  1. Type: Make foo dynamic
  2. WP Admin → ACF → Sync changes
  3. Edit the page, fill fields, save
Rule: never skip Phase 1. Going straight to ACF means fighting pixel-perfect and ACF at the same time.

ACF + sync

All field group changes go through acf-json/. You manually click "Sync changes" in WP Admin to apply.

The timestamp mechanism

  1. Claude writes / edits acf-json/<group>.json
  2. Claude bumps the "modified" field to current unix time
  3. ACF Pro sees the new timestamp on next admin page load
  4. WP Admin shows "X field groups have changes available"
  5. You click "Sync changes" → fields updated in DB, appear in editor

Images & SVG

Native HTML <img> and <picture>. Image plugin handles WebP. Every <img> needs width + height for CLS.

ACF image — single

<?php $img = get_sub_field('image'); if ($img) : ?>
<img
    src="<?= esc_url($img['url']) ?>"
    alt="<?= esc_attr($img['alt']) ?>"
    width="<?= (int) $img['width'] ?>"
    height="<?= (int) $img['height'] ?>"
    class="w-full h-auto"
    loading="lazy"
    decoding="async"
>
<?php endif; ?>

ACF image + mobile companion → <picture>

<?php
$img = get_sub_field('image');
$mobile = get_sub_field('mobile_image');
if ($img) :
?>
<picture>
    <?php if ($mobile) : ?>
        <source
            srcset="<?= esc_url($mobile['url']) ?>"
            media="(max-width: 767px)"
            width="<?= (int) $mobile['width'] ?>"
            height="<?= (int) $mobile['height'] ?>">
    <?php endif; ?>
    <img
        src="<?= esc_url($img['url']) ?>"
        alt="<?= esc_attr($img['alt']) ?>"
        width="<?= (int) $img['width'] ?>"
        height="<?= (int) $img['height'] ?>"
        loading="lazy">
</picture>
<?php endif; ?>

Static theme image — auto width/height

<?php aiims_img('hero/banner.jpg', 'Alt text', 'w-full h-auto'); ?>

SVG that needs theming (currentColor)

<?php $svg = get_sub_field('icon_svg'); if ($svg) : ?>
    <div class="w-12 h-12 text-primary">
        <?= wp_kses($svg, aiims_svg_kses()) ?>
    </div>
<?php endif; ?>

Pixel-perfect spacing decisions

  • Within 2px of a Tailwind token → use the token.
  • 3-5px off → ask before deciding.
  • >5px or hard requirement → arbitrary [73px] + flag deviation.

Anti-patterns — don't do these

Common ways to get into trouble. Each one looks fine in the moment and bites you later.

❌ Editing theme/functions.php with custom code

Do instead: add to theme/inc/custom-functions.php. Keeps underscoretw upstream clean and isolates project code in one place.

❌ Adding fields in WP Admin's ACF UI directly

Do instead: ask Claude to update acf-json/<group>.json, sync from there. Otherwise schema lives in the database, can't be code-reviewed, can't be moved between environments.

❌ Skipping Phase 1 (jumping straight to ACF)

Do instead: always build static first, verify pixel-perfect, then make dynamic. Otherwise you debug ACF AND markup at the same time.

❌ <img> tags without width and height

Do instead: ACF arrays already include width/height — use them. Static images use aiims_img() which reads them from disk. Without these → CLS penalty in Lighthouse.

❌ Using display: none to hide content on mobile

Do instead: restructure (different order, smaller version, accordion, drawer). Hidden content still ships bytes, hurts a11y, and is usually a sign the design wasn't planned for mobile.

❌ Hardcoded hex colors in section markup

Do instead: use brand tokens (bg-primary, text-secondary). One-off arbitrary values are OK if you flag them as deviations. 2+ uses of a new color → register it via tailwind-theme-sync.

❌ Static image paths in dynamic ACF sections

Do instead: every section image becomes an ACF upload after Phase 2. aiims_img() with a static path is only for theme-wide assets (header logo, decorative shapes that ship with the theme).

❌ Regenerating ACF field key values when editing a layout

Do instead: preserve the existing key when changing name or label. Changing the key orphans saved data.

❌ Adding new page templates instead of using flexible content

Do instead: add a layout to flexible_sections (use Add a section: <name>). New page templates only when the page truly needs a different structure (use /create-template for that).

❌ Skipping escapes on dynamic output

Do instead: esc_html for text, esc_attr for attributes, esc_url for links, wp_kses_post for rich text. Skipping escapes = XSS hole.

❌ Comments explaining what obvious code does

Do instead: code should be readable. Comments only when the WHY isn't obvious. // Get the heading above $heading = get_sub_field('heading'); is noise.

Git tips

What to commit

✓ Commit these

  • Everything in theme/ source files (PHP, CSS source)
  • theme/acf-json/*.json ⭐ schema source of truth
  • briefs/ — section briefs are useful project history
  • .claude/ — your config travels with the project
  • README.md
  • Tailwind source: tailwind/, tailwind.css
  • javascript/script.js
  • package.json + package-lock.json

✗ Ignore these

  • node_modules/
  • theme/style.css (compiled — built from Tailwind source)
  • theme/style-editor.css (compiled)
  • theme/js/*.min.js (compiled — built from javascript/)
  • tailwind-intellisense.css (dev artifact)
  • theme/assets/images/<section>/*.webp (auto-generated by image plugin)
  • Generated zip files (*.zip)
  • Editor cruft (.DS_Store, Thumbs.db)

Branching tips

  • One branch per section build (section/home-hero) makes review easy
  • Commit ACF JSON changes alongside the section file — they go together
  • For multi-environment deploys (dev/staging/live), the acf-json/ folder syncs the schema; just sync ACF on each environment after pulling
  • Don't commit theme/style.css — it's built. Run npm run watch on each environment instead

When things break

bg-X Tailwind class doesn't work›
Token's not in @theme. Type add brand color X #hex.
Section won't render›
Check theme/templates/parts/section-<dashes-name>.php exists. File uses dashes, ACF layout name uses underscores. They must match.
ACF fields don't show in editor›
Sync hasn't been run. WP Admin → Custom Fields → Field Groups → "Sync changes". Or check the location rule.
WebP not being served›
Image optimization plugin not active. Install ShortPixel / EWWW / Imagify and configure WebP delivery.
Skills / commands not loading›
.claude/ folder must be at workspace root with leading dot. ren claude .claude if needed.
Tailwind class in markup but not styled›
npm run watch not running. Open a terminal in workspace root and run it. Hard-refresh.
Force a new ACF sync notice›
Type bump acf timestamps.
Briefs vs Add a section›
"Build X from Figma" = Claude makes the brief. "@briefs/x.md Figma" = you wrote it. "Add a section: x" = scaffold only.
White screen / PHP fatal after I added something›
Probably syntax error or duplicate function in a file you added. Enable WP_DEBUG in wp-config.php to see the error. If WP Admin is down, edit theme/inc/custom-functions.php directly to comment out the latest require_once.
Switching between Cowork and Claude Code›
Both read the same .claude/. Conversation history is separate. To hand off, ask the new instance "what was the last section we worked on?" — Claude reads the section files to figure out state.
